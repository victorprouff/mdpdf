Mets à jour la documentation du projet en fonction des changements récents.

## Étapes

1. Lis le diff git des changements non commités (`git diff`) et les commits récents (`git log --oneline -10`) pour comprendre ce qui a changé.

2. Lis les fichiers actuels :
   - `CLAUDE.md`
   - `README.md`
   - `package.json`

3. Mets à jour **CLAUDE.md** si nécessaire :
   - Structure du projet (nouveaux fichiers/dossiers)
   - Nouvelles dépendances
   - Nouvelles conventions de code
   - Changements d'architecture

4. Mets à jour **README.md** si nécessaire :
   - Nouvelles fonctionnalités dans la section Fonctionnalités
   - Nouvelles options CLI dans le tableau Options
   - Nouveaux exemples d'utilisation
   - Mise à jour du front matter supporté
   - Cocher les items dans la Todo list si complétés

5. Ne fais **aucune modification** si rien n'a changé. Dis simplement que les docs sont à jour.

6. Résume les modifications apportées.
